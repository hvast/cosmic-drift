# 🚀 Cosmic Drift 部署指南

## 部署架构

```
前端 (Vercel)  →  后端 (Railway/Render)  →  MySQL 数据库
```

---

## 方案一：前端 Vercel + 后端 Railway（推荐）

### 1. 部署后端到 Railway

Railway 是一个现代化的部署平台，支持 Node.js 和 MySQL。

#### 步骤：

1. **注册 Railway**
   - 访问 https://railway.app
   - 使用 GitHub 账号登录

2. **创建新项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择你的 cosmicDrift 仓库

3. **配置后端服务**
   - Railway 会自动检测到 Node.js 项目
   - 设置根目录为 `backend`
   - 构建命令：`npm install && npm run build`
   - 启动命令：`npm start`

4. **添加 MySQL 数据库**
   - 在项目中点击 "New"
   - 选择 "Database" → "MySQL"
   - Railway 会自动创建数据库并提供连接信息

5. **配置环境变量**
   在 Railway 项目设置中添加：
   ```
   NODE_ENV=production
   PORT=3001
   
   # 数据库配置（Railway 会自动注入）
   DB_HOST=${{MYSQLHOST}}
   DB_PORT=${{MYSQLPORT}}
   DB_USER=${{MYSQLUSER}}
   DB_PASSWORD=${{MYSQLPASSWORD}}
   DB_NAME=${{MYSQLDATABASE}}
   
   # JWT 密钥
   JWT_SECRET=your-production-jwt-secret-change-this
   
   # 通义千问 API
   QWEN_API_KEY=your-qwen-api-key
   
   # CORS 配置
   CORS_ORIGIN=https://your-frontend-domain.vercel.app
   ```

6. **运行数据库迁移**
   - 在 Railway 控制台中打开终端
   - 运行：`npm run migrate`

7. **获取后端 URL**
   - Railway 会自动生成一个 URL，如：`https://cosmicDrift-production.up.railway.app`
   - 记下这个 URL，前端需要用到

---

### 2. 部署前端到 Vercel

#### 步骤：

1. **安装 Vercel CLI**（可选）
   ```bash
   npm install -g vercel
   ```

2. **准备前端代码**
   
   在 `frontend/package.json` 中确认构建脚本：
   ```json
   {
     "scripts": {
       "build": "react-scripts build"
     }
   }
   ```

3. **通过 Vercel 网站部署**
   
   a. 访问 https://vercel.com
   b. 使用 GitHub 账号登录
   c. 点击 "Add New Project"
   d. 选择你的 cosmicDrift 仓库
   e. 配置项目：
      - Framework Preset: `Create React App`
      - Root Directory: `frontend`
      - Build Command: `npm run build`
      - Output Directory: `build`
   
   f. 添加环境变量：
      ```
      REACT_APP_API_URL=https://your-backend-url.railway.app
      ```
   
   g. 点击 "Deploy"

4. **或使用 CLI 部署**
   ```bash
   cd frontend
   vercel
   ```
   
   按提示操作，设置环境变量。

5. **配置自定义域名**（可选）
   - 在 Vercel 项目设置中添加自定义域名
   - 更新后端的 CORS_ORIGIN 环境变量

## 方案二：前端 Vercel + 后端 Render

Render 是另一个优秀的部署平台，免费套餐也很不错。

### 1. 部署后端到 Render

1. **注册 Render**
   - 访问 https://render.com
   - 使用 GitHub 账号登录

2. **创建 Web Service**
   - 点击 "New +" → "Web Service"
   - 连接你的 GitHub 仓库
   - 配置：
     - Name: `cosmicDrift-backend`
     - Root Directory: `backend`
     - Environment: `Node`
     - Build Command: `npm install && npm run build`
     - Start Command: `npm start`

3. **创建 MySQL 数据库**
   - Render 免费套餐不包含 MySQL
   - 可以使用 PlanetScale（免费 MySQL）或 Railway

4. **配置环境变量**
   在 Render 的 Environment 设置中添加所有必要的环境变量

5. **部署**
   - Render 会自动构建和部署
   - 获取生成的 URL

### 2. 部署前端到 Vercel
   同方案一

## 方案三：全栈部署到云服务器

如果你有云服务器（阿里云、腾讯云等），可以全部部署到一台服务器上。

### 部署步骤：

1. **准备服务器**
   - Ubuntu 20.04+ 或 CentOS 7+
   - 安装 Node.js 18+、MySQL 8.0+、Nginx

2. **部署后端**
   ```bash
   # 克隆代码
   git clone <your-repo>
   cd cosmicDrift/backend
   
   # 安装依赖
   npm install
   
   # 配置环境变量
   cp .env.example .env
   nano .env
   
   # 运行迁移
   npm run migrate
   
   # 使用 PM2 启动
   npm install -g pm2
   pm2 start npm --name "cosmicDrift-backend" -- start
   pm2 save
   pm2 startup
   ```

3. **构建前端**
   ```bash
   cd ../frontend
   npm install
   npm run build
   ```

4. **配置 Nginx**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       # 前端
       location / {
           root /path/to/cosmicDrift/frontend/build;
           try_files $uri /index.html;
       }
       
       # 后端 API
       location /api {
           proxy_pass http://localhost:3001;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
       
       # 上传的文件
       location /uploads {
           proxy_pass http://localhost:3001;
       }
   }
   ```

5. **配置 SSL（推荐）**
   ```bash
   # 使用 Let's Encrypt
   sudo apt install certbot python3-certbot-nginx
   sudo certbot --nginx -d your-domain.com
   ```

## 常见问题

### 1. CORS 错误
**问题**：前端无法访问后端 API

**解决**：
- 确保后端 `CORS_ORIGIN` 环境变量设置为前端域名
- 检查后端 CORS 中间件配置

### 2. 数据库连接失败
**问题**：后端无法连接数据库

**解决**：
- 检查数据库环境变量是否正确
- 确认数据库服务正在运行
- 检查防火墙规则

### 3. 图片上传失败
**问题**：上传的图片无法访问

**解决**：
- 确保 `backend/uploads` 目录存在且有写权限
- 检查静态文件服务配置
- 考虑使用对象存储（OSS、S3）

### 4. 环境变量不生效
**问题**：配置的环境变量没有生效

**解决**：
- 前端环境变量必须以 `REACT_APP_` 开头
- 修改环境变量后需要重新构建
- Vercel 需要在项目设置中配置环境变量

### 5. 构建失败
**问题**：部署时构建失败

**解决**：
- 检查 Node.js 版本是否匹配
- 确认所有依赖都在 `package.json` 中
- 查看构建日志找出具体错误

## 性能优化建议

### 前端优化
1. 启用 Vercel 的 CDN 加速
2. 配置图片优化
3. 启用 Gzip 压缩
4. 使用代码分割（React.lazy）

### 后端优化
1. 使用 Redis 缓存（可选）
2. 配置数据库连接池
3. 启用 API 响应压缩
4. 添加请求限流

### 数据库优化
1. 添加适当的索引
2. 定期备份数据
3. 监控查询性能

## 成本估算

### Railway（推荐新手）
- 免费套餐：$5 免费额度/月
- 付费套餐：按使用量计费，约 $5-20/月

### Render
- 免费套餐：有限制但够用
- 付费套餐：$7/月起

### Vercel（前端）
- 免费套餐：个人项目完全够用
- Pro 套餐：$20/月（团队使用）

### 云服务器
- 阿里云/腾讯云：约 ¥100-300/月
- 需要自己维护

**对于个人项目/学习**：
- 前端：Vercel（免费）
- 后端：Railway（$5 免费额度）

- 前端：Vercel Pro
- 后端：云服务器 + PM2
- 数据库：云数据库（RDS）
