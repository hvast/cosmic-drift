# 项目结构说明

## 📁 根目录结构

```
cosmic-drift/
├── 📂 frontend/              # React 前端应用
├── 📂 backend/               # Node.js 后端应用
├── 📂 scripts/               # 启动和工具脚本
├── 📂 docs/                  # 项目文档
├── 📂 .kiro/                 # Kiro 规格文档
├── 📄 启动开发环境.bat        # 快速启动入口（双击运行）
├── 📄 package.json           # 根项目配置
└── 📄 README.md              # 项目说明
```

## 🚀 启动脚本 (scripts/)

### 用户脚本
- **快速启动.bat** - 交互式菜单，选择启动方式
- **start-dev.bat** - 启动完整开发环境（前端+后端）
- **start-dev.ps1** - PowerShell 版本启动脚本
- **install-all.bat** - 安装所有依赖

### 使用方法
```bash
# 方式 1: 双击根目录的"启动开发环境.bat"
# 方式 2: 命令行运行
scripts\start-dev.bat
```

- **开发规范.md** - 代码规范和最佳实践

## 📋 规格文档 (.kiro/specs/cosmic-drift/)

```
.kiro/specs/cosmic-drift/
├── requirements.md         # 需求文档（EARS 格式）
├── design.md              # 系统设计文档

```

### 文档说明

- **requirements.md**: 包含 10 个核心需求，每个需求都有用户故事和验收标准
- **design.md**: 详细的系统架构、组件设计、数据模型和技术选型
- **tasks.md**: 80+ 个具体的开发任务，按模块组织

## 🔑 关键文件说明

### 根目录

- **启动开发环境.bat**: 主启动入口，提供交互式菜单
- **package.json**: 定义了所有 npm 脚本和依赖
- **README.md**: 项目总览和快速开始指南

### 配置文件

- **frontend/.env**: 前端环境变量（API URL 等）
- **backend/.env**: 后端环境变量（数据库、API 密钥等）
- **tsconfig.json**: TypeScript 配置
- **.eslintrc.json**: ESLint 代码检查配置
- **.prettierrc**: Prettier 代码格式化配置

## 📦 依赖管理

### 根目录依赖
- **concurrently**: 同时运行前后端开发服务器

### 前端主要依赖
- **react**: UI 框架
- **three.js**: 3D 可视化
- **fabric.js**: 画布绘图
- **tailwindcss**: 样式系统
- **framer-motion**: 动画库
- **socket.io-client**: WebSocket 客户端

### 后端主要依赖
- **express**: Web 框架
- **socket.io**: WebSocket 服务器
- **pg**: PostgreSQL 客户端
- **redis**: Redis 客户端
- **bcrypt**: 密码加密
- **jsonwebtoken**: JWT 认证
- **axios**: HTTP 客户端（调用 AI API）

## 🗄️ 数据库表结构

### 主要表
- **users**: 用户信息
- **creatures**: 生物档案
- **conversations**: 对话记录
- **messages**: 消息内容
- **affinity_scores**: 契合度分数
- **cosmic_log_entries**: 宇宙日志
- **daily_statuses**: 每日状态

详细 schema 见 `backend/migrations/` 目录

## 🌐 端口配置

- **前端**: http://localhost:3000
- **后端**: http://localhost:5000
- **PostgreSQL**: localhost:5432
- **Redis**: localhost:6379

## 📝 开发工作流

1. **启动项目**: 双击 `启动开发环境.bat`
2. **选择模式**: 选择启动完整环境或单独前/后端
3. **开发**: 修改代码，自动热重载
4. **测试**: 在浏览器查看效果
5. **提交**: 遵循 Git 提交规范
