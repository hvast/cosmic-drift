# 对话功能快速启动指南

## 前置条件

1. 已完成项目基础设置（数据库、依赖安装）
2. 已配置OpenAI API Key

## 步骤1: 运行数据库迁移

在项目根目录执行：

```bash
# Windows
cd backend
npm run migrate

# 或者直接运行
npx ts-node backend/migrations/run-migrations.ts
```

这将创建 `conversations` 和 `messages` 两个表。

## 步骤2: 配置环境变量

确保 `backend/.env` 中有以下配置：

```env
OPENAI_API_KEY=your-openai-api-key-here
```

如果没有OpenAI API Key，系统会使用降级方案（预设回复）。

## 步骤3: 启动服务

### 启动后端
```bash
cd backend
npm run dev
```

### 启动前端
```bash
cd frontend
npm start
```

## 步骤4: 使用对话功能

### 方式1: 从星图开始对话
1. 访问 http://localhost:3000/galaxy
2. 点击任意生物节点
3. 在档案卡片中点击"开始对话"按钮
4. 进入对话界面

### 方式2: 查看对话列表
1. 点击导航栏的"对话"
2. 查看所有历史对话
3. 点击任意对话卡片继续聊天

### 方式3: 直接访问
访问 http://localhost:3000/chat/:creatureId

## 功能演示

### 发送消息
1. 在输入框输入消息（最多500字符）
2. 点击"发送"按钮
3. 等待AI生成回复（通常2-5秒）
4. 查看生物的回复和情绪值变化

### 观察情绪变化
- 发送积极的消息（如"你好美"、"我喜欢你"）→ 情绪值上升
- 发送消极的消息（如"好无聊"、"不开心"）→ 情绪值下降
- 情绪值影响生物的回复风格

### 查看对话历史
- 所有消息自动保存
- 刷新页面后对话历史依然存在
- 可以在对话列表页查看所有对话

## 测试建议

### 测试不同性格的生物
创建多个不同性格的生物，观察它们的回复差异：
- 温柔型生物 → 回复更柔和
- 冷淡型生物 → 回复更简短
- 好奇型生物 → 会提出问题

### 测试情绪系统
1. 与一个生物持续发送积极消息
2. 观察情绪值从50逐渐上升到80+
3. 注意回复风格的变化

### 测试对话记忆
1. 提到某个话题（如"星空"）
2. 几条消息后再次提到
3. 观察AI是否能联系上下文

## 常见问题

### Q: AI回复很慢或超时
A: 
- 检查网络连接
- 检查OpenAI API Key是否有效
- 查看后端控制台是否有错误
- 如果API不可用，系统会自动使用降级回复

### Q: 情绪值不变化
A:
- 检查后端日志，确认情绪计算逻辑执行
- 尝试使用明确的积极/消极词汇
- 刷新页面重新加载

### Q: 看不到对话历史
A:
- 确认已登录
- 检查数据库迁移是否成功
- 查看浏览器控制台是否有错误

### Q: 生物回复不符合角色
A:
- 检查生物的档案信息是否完整
- OpenAI API可能需要几次对话才能稳定角色
- 可以尝试重新开始对话
