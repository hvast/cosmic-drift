## 🎯 快速启动

### 方式一：使用启动脚本（推荐）

**Windows 用户：**
```bash
# 双击运行
启动开发环境.bat

# 或在命令行中运行
.\启动开发环境.bat
```

这会显示一个菜单，你可以选择：
- `[1]` 启动完整开发环境（前端+后端）
- `[2]` 仅启动前端
- `[3]` 仅启动后端
- `[4]` 安装所有依赖
- `[5]` 运行数据库迁移

### 方式二：使用 npm 命令

```bash
# 一键启动前端和后端（会自动并行运行）
npm run dev

# 或分别启动
npm run dev:backend   # 仅启动后端
npm run dev:frontend  # 仅启动前端
```

## 📝 分步启动（第一次运行）

### 1. 检查前置要求

确保已安装：
- ✅ Node.js 18+ （运行 `node -v` 检查）
- ✅ MySQL 8.0+ （运行 `mysql --version` 检查）
- ✅ Redis 6+ （可选，用于缓存）

### 2. 配置数据库

```bash
# 1. 登录 MySQL
mysql -u root -p

# 2. 创建数据库
CREATE DATABASE cosmic_drift CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 3. 退出
exit;
```

### 3. 配置环境变量

**后端配置 (`backend/.env`)：**
```bash
# 复制示例配置
cd backend
cp .env.example .env

# 编辑 .env 文件，修改以下配置：
DB_HOST=localhost
DB_PORT=3306
DB_NAME=cosmic_drift
DB_USER=root
DB_PASSWORD=你的MySQL密码

# Redis 配置（可选）
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=你的Redis密码

# 通义千问 API（可选，用于AI功能）
QWEN_API_KEY=你的通义千问 API 密钥
```

**前端配置 (`frontend/.env`)：**
```bash
cd frontend
# 前端默认配置通常不需要修改
REACT_APP_API_URL=http://localhost:3001
```

### 4. 安装依赖

```bash
# 方式一：使用脚本（推荐）
.\启动开发环境.bat
# 选择 [4] 安装所有依赖

# 方式二：手动安装
npm install              # 安装根目录依赖
cd backend && npm install   # 安装后端依赖
cd ../frontend && npm install  # 安装前端依赖
```

### 5. 运行数据库迁移

```bash
# 方式一：使用脚本
.\启动开发环境.bat
# 选择 [5] 运行数据库迁移

# 方式二：手动运行
npm run migrate
```

### 6. 启动项目

```bash
# 使用脚本启动
.\启动开发环境.bat
# 选择 [1] 启动完整开发环境

# 或使用 npm 命令
npm run dev
```

### 7. 访问应用

- 🎨 **前端**: http://localhost:3000
- 🔧 **后端 API**: http://localhost:3001
- 📊 **健康检查**: http://localhost:3001/health

## 🐛 调试方法

### 1. 后端调试

#### 方式一：使用 VS Code 调试器（推荐）

创建 `.vscode/launch.json`：
```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug Backend",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "dev"],
      "cwd": "${workspaceFolder}/backend",
      "console": "integratedTerminal",
      "skipFiles": ["<node_internals>/**"]
    }
  ]
}
```

使用方法：
1. 打开 VS Code
2. 按 `F5` 或点击 "运行和调试"
3. 选择 "Debug Backend"
4. 在代码中设置断点（点击行号左侧）

#### 方式二：使用 console.log 调试

在代码中添加日志：
```typescript
// backend/src/controllers/AuthController.ts
register = async (req: Request, res: Response) => {
  console.log('📝 注册请求数据:', req.body);
  // ... 其他代码
}
```

查看日志：
```bash
cd backend
npm run dev
# 日志会实时显示在终端
```

#### 方式三：查看数据库日志

已启用的查询日志（在 `backend/src/config/database.ts`）：
```typescript
console.log('Executed query', {
  text: mysqlQuery,
  duration,
  rows: Array.isArray(rows) ? rows.length : 0
});
```

### 2. 前端调试

#### 使用浏览器开发者工具

1. **打开开发者工具**

   - 查看 console.log 输出

3. **Network 面板**
   - 查看 API 请求


## ❓ 常见问题

### 问题 1: 端口被占用

**错误信息:**
```
Error: listen EADDRINUSE: address already in use :::3001
```

**解决方案:**
```bash
# Windows: 查找占用端口的进程
netstat -ano | findstr :3001

# 杀死进程（替换 PID 为实际进程ID）
taskkill /PID <PID> /F

# 或者修改端口配置
# 编辑 backend/.env
PORT=3002
```

### 问题 2: MySQL 连接失败

**错误信息:**
```
❌ MySQL connection error: ER_ACCESS_DENIED_ERROR
```

**解决方案:**
```bash
# 1. 检查 backend/.env 配置
DB_USER=root
DB_PASSWORD=正确的密码

# 2. 测试 MySQL 连接
mysql -u root -p

# 3. 确保数据库存在
mysql> SHOW DATABASES;
mysql> USE cosmic_drift;
```

### 问题 3: 前端无法连接后端

**错误信息:**
```
Network Error: Failed to connect to server
```

**解决方案:**
```bash
# 1. 确认后端正在运行
cd backend
npm run dev

# 2. 检查后端地址配置
# frontend/.env
REACT_APP_API_URL=http://localhost:3001

# 3. 检查 CORS 配置
# backend/src/index.ts 应该包含:
app.use(cors());
```

### 问题 4: 注册或登录失败

**错误信息:**
```
INTERNAL_SERVER_ERROR
```

**调试步骤:**
```bash
# 1. 查看后端日志
cd backend
npm run dev
# 观察控制台输出

# 2. 检查数据库表是否存在
mysql -u root -p cosmic_drift
mysql> SHOW TABLES;
mysql> DESCRIBE users;

# 3. 测试 API
curl -X POST http://localhost:3001/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@test.com","password":"Test1234"}'
```

### 问题 5: 依赖安装失败

**错误信息:**
```
npm ERR! code ERESOLVE
```

**解决方案:**
```bash
# 清除缓存并重新安装
npm cache clean --force
rm -rf node_modules package-lock.json
npm install

# 或使用 --legacy-peer-deps
npm install --legacy-peer-deps
```

## 📊 性能监控

### 后端性能监控

```typescript
// 添加请求计时中间件
app.use((req, res, next) => {
  const start = Date.now();
  res.on('finish', () => {
    const duration = Date.now() - start;
    console.log(`${req.method} ${req.path} - ${duration}ms`);
  });
  next();
});
```

### 数据库查询优化

查询日志已自动记录：
```
Executed query { text: '...', duration: 15, rows: 10 }
```

如果查询时间过长（> 100ms），考虑：
- 添加索引
- 优化查询语句
- 使用缓存

## 🎓 调试技巧

### 1. 分层调试

**前端 → 后端 → 数据库**

```
用户操作
  ↓
前端组件 (console.log)
  ↓
API 调用 (Network 面板)
  ↓
后端控制器 (console.log)
  ↓
服务层 (console.log)
  ↓
数据库查询 (查询日志)
```

### 2. 使用断点

```typescript
// 在关键位置设置断点
debugger; // 代码会在此处暂停
```

### 3. 错误追踪

```typescript
try {
  // 可能出错的代码
} catch (error) {
  console.error('完整错误信息:', error);
  console.error('错误堆栈:', error.stack);
  // 发送到日志服务
}
```
