# AI对话系统实现文档

## 概述

本模块实现了星际漂流计划的核心功能：AI角色对话系统。用户可以与他们创建或遇见的数字生命进行符合角色背景的对话，生物会根据自己的性格、背景故事和当前情绪值做出独特的回应。

## 功能特性

### 1. 角色化对话
- AI根据生物的物种、性格、栖息地和背景故事生成回复
- 回复风格符合生物的角色设定
- 保持诗意、神秘的语言风格

### 2. 情绪系统
- 每个生物有0-100的情绪值
- 情绪值影响回复的温度（冷淡/平静/温暖）
- 用户的积极/消极语气会影响生物的情绪值
- 情绪变化实时反映在UI上

### 3. 对话记忆
- 临时记忆：漂流状态下保留最近10条消息作为上下文
- 长期记忆：认养后可保存完整对话历史（待实现）
- AI回复时会参考对话历史

### 4. 对话管理
- 查看所有对话列表
- 继续之前的对话
- 删除对话记录
- 显示对话统计（消息数、最后活跃时间）

## 技术实现

### 后端服务

#### 1. ConversationAIService
**位置**: `backend/src/services/ConversationAIService.ts`

**功能**:
- 使用通义千问 Qwen-Plus 生成符合角色的回复
- 构建包含生物背景和性格的系统提示词
- 根据情绪值调整回复风格
- 分析用户消息并计算情绪变化
- 提供降级方案（AI服务不可用时）

**核心方法**:
```typescript
generateResponse(userMessage: string, context: ConversationContext): Promise<AIResponse>
```

#### 2. ConversationService
**位置**: `backend/src/services/ConversationService.ts`

**功能**:
- 协调对话流程
- 管理对话和消息的创建
- 更新生物情绪值
- 获取对话历史

**核心方法**:
```typescript
sendMessage(request: SendMessageRequest): Promise<SendMessageResponse>
getConversation(userId: string, creatureId: string)
getUserConversations(userId: string)
deleteConversation(conversationId: string, userId: string)
```

#### 3. ConversationRepository
**位置**: `backend/src/repositories/ConversationRepository.ts`

**功能**:
- 对话和消息的数据库操作
- CRUD操作
- 查询和分页

### 数据模型

#### Conversation（对话）
```typescript
{
  id: string;
  userId: string;
  creatureId: string;
  memoryType: 'temporary' | 'longterm';
  affinityScore: number;
  startedAt: Date;
  lastMessageAt: Date;
  messageCount: number;
  isPublished: boolean;
}
```

#### Message（消息）
```typescript
{
  id: string;
  conversationId: string;
  senderId: string;
  senderType: 'user' | 'creature';
  content: string;
  timestamp: Date;
  sentiment?: {
    score: number;
    tone: 'positive' | 'neutral' | 'negative';
  };
}
```

### API端点

#### 发送消息
```
POST /api/conversations/message
Body: { creatureId: string, content: string }
Response: { userMessage, creatureMessage, conversation, emotionValue }
```

#### 获取对话历史
```
GET /api/conversations/:creatureId
Response: { conversation, messages }
```

#### 获取对话列表
```
GET /api/conversations
Response: [{ conversation, creature, lastMessage }]
```

#### 删除对话
```
DELETE /api/conversations/:conversationId
```

## 使用流程

### 1. 开始对话
```
用户在星图中点击生物 → 查看档案 → 点击"开始对话"
→ 跳转到 /chat/:creatureId
→ 系统加载生物信息和历史对话
```

### 2. 发送消息
```
用户输入消息 → 点击发送
→ 前端调用 conversationService.sendMessage()
→ 后端创建/获取对话
→ 保存用户消息
→ AI生成回复（基于角色和上下文）
→ 更新生物情绪值
→ 保存AI回复
→ 返回完整响应
→ 前端更新UI
```

### 3. 查看对话列表
```
用户点击导航栏"对话"
→ 跳转到 /conversations
→ 加载所有对话
→ 显示对话卡片列表
→ 点击卡片进入对话
```

## AI提示词设计

系统提示词包含以下信息：
1. 生物的基本信息（名称、物种、性格、栖息地、背景故事）
2. 当前情绪状态和对应的回复风格
3. 对话规则（保持角色、简洁诗意、不使用emoji等）

示例：
```
你是一个名为"星尘"的数字生命体。

【你的基本信息】
- 物种：流光精灵
- 性格特征：温柔、好奇、略带忧郁
- 栖息地：紫色星云的边缘
- 背景故事：在宇宙深处漂流了千年...

【你的当前状态】
- 情绪值：65/100
- 你现在处于平静的状态，回复会保持礼貌但不过分热情。

【对话规则】
1. 始终保持你的性格特征
2. 回复要简洁而富有诗意，通常在1-3句话之间
3. 不要使用emoji，保持神秘而优雅的语气
...
```

## 情绪系统

### 情绪值范围
- 0-29: 冷淡（回复简短克制）
- 30-69: 平静（礼貌但不热情）
- 70-100: 温暖（友好开放）

### 情绪变化规则
用户消息中的关键词影响情绪值：
- 积极词汇（喜欢、爱、美好等）：+2
- 消极词汇（讨厌、难过、孤独等）：-1
- 每次变化限制在 -5 到 +5 之间
- 情绪值范围限制在 0-100

## 性能优化

1. **响应时间**
   - AI生成目标：< 3秒
   - 使用通义千问 Qwen-Plus，temperature=0.85
   - max_tokens=300（控制回复长度）

2. **上下文管理**
   - 只保留最近10条消息作为上下文
   - 减少token消耗
   - 保持对话连贯性

3. **降级方案**
   - AI服务不可用时使用预设回复
   - 确保用户体验不中断
