# 验证规则更新

## 修改内容

### 背景故事现在可以为空 ✅

**之前：**
- ❌ 背景故事必填
- ❌ 至少需要10个字符
- ❌ 报错："背景故事至少需要10个字符"

**现在：**
- ✅ 背景故事可以为空
- ✅ 如果为空，后端会使用默认值："这个生命的故事尚未被书写..."
- ✅ 不再有字符限制错误

## 当前验证规则

### 必填字段（3个）

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 名称 | 必填，非空 | 请输入生物名称 |
| 物种 | 必填，非空 | 请输入或选择物种 |
| 栖息地 | 必填，非空 | 请输入或选择栖息地 |

### 可选字段（3个）

| 字段 | 规则 | 默认值 |
|------|------|--------|
| 性格特征 | 可选，最多6个 | 空数组 [] |
| 背景故事 | 可选，无限制 | "这个生命的故事尚未被书写..." |
| 情绪值 | 0-100 | 50 |

## 修改的文件

### 1. 前端验证
**文件：** `frontend/src/components/CreatureProfileCard.tsx`

```typescript
// 移除了背景故事的验证
// 只验证名称、物种、栖息地
```

### 2. 后端验证器
**文件：** `backend/src/validators/creatureValidator.ts`

```typescript
// 允许空字符串
story: Joi.string().min(0).max(1000).allow('').optional()
```

### 3. 后端服务
**文件：** `backend/src/services/CreatureService.ts`

```typescript
// 检查完整性时不再要求 story
const hasCompleteCustomization = 
  request.userCustomization && 
  request.userCustomization.name &&
  request.userCustomization.species &&
  request.userCustomization.habitat;
  // story 不再是必需的

// 如果 story 为空，使用默认值
backstory: request.userCustomization!.story! || '这个生命的故事尚未被书写...'
```

## 使用示例

### 最简创建（只填3个必填字段）

```
名称：星之灵 (点击🎲随机生成)
物种：星云游者
栖息地：星云深处
性格特征：(留空)
背景故事：(留空)
情绪值：50 (默认)
```

点击"保存 SAVE" → ✅ 成功创建！

**生成的档案：**
```json
{
  "name": "星之灵",
  "species": "星云游者",
  "personality": [],
  "habitat": "星云深处",
  "backstory": "这个生命的故事尚未被书写...",
  "emotionValue": 50
}
```

### 完整创建（填写所有字段）

```

性格特征：温柔、神秘、智慧
栖息地：月光笼罩的星域
背景故事：诞生于遥远的月光星云，带着温柔的歌声在宇宙中漂流...
情绪值：75
```

点击"保存 SAVE" → ✅ 成功创建！
