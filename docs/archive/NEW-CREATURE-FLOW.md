# 🎨 新的生物创建流程

## 流程改进

### 之前的流程
1. 绘制/上传图片
2. 点击"生成档案" → 直接调用 AI
3. 显示结果

### 现在的流程
1. 绘制/上传图片
2. 点击"提交证件照 📸"
3. **预览证件照卡片** ⭐ 新增
4. 选择：
   - ✍️ **自己编写** - 手动填写背景故事
   - 🤖 **AI 生成** - 让 AI 创作
5. 编辑/完善档案
6. 发布到星系

## 核心改进点

### 1. 证件照预览卡片
- 用户可以先看到自己绘制的图片
- 在一个精美的卡片中展示
- 给用户思考的时间

### 2. 用户选择权
- **自己编写**：跳过 AI，直接进入编辑模式，所有字段为空
- **AI 生成**：调用后端 API，AI 生成完整档案

### 3. 灵活的编辑
- 无论选择哪种方式，都可以在编辑页面修改
- 手动编辑模式下，保存时才会创建生物记录
- AI 生成模式下，生物已创建，保存时更新记录

## 技术实现

### 新增步骤
```typescript
type CreationStep = 'method' | 'draw' | 'upload' | 'preview' | 'generating' | 'edit';
```

### 关键函数

**handleSubmitPhoto()**
- 从绘制/上传步骤进入预览步骤
- 不调用任何 API，只是状态切换

**handleManualEdit()**
- 创建一个临时的空档案模板
- ID 以 'temp-' 开头，标识为未保存状态
- 进入编辑模式

**handleAIGenerate()**
- 调用后端 API 生成完整档案
- 显示加载动画
- 成功后进入编辑模式

**handleUpdateProfile()**
- 检查是否为临时 ID
- 如果是临时 ID：调用创建 API
- 如果不是：调用更新 API

## 用户体验优化

### 1. 心理缓冲
- 用户绘制完成后不会立即被 AI 生成打断
- 有时间欣赏自己的作品
- 可以决定是否需要 AI 帮助

### 2. 控制感
- 用户可以选择完全自主创作
- 也可以选择 AI 辅助
- 两种方式都能达到目标

### 3. 灵活性
- 选择 AI 生成后仍可修改
- 选择手动编写也可以参考 AI 的建议（未来可扩展）

## 注意事项

### 手动编辑模式
- 临时 ID 格式：`temp-{timestamp}`
- 保存前不会创建数据库记录
- 如果用户中途退出，不会留下垃圾数据

### AI 生成模式
- 立即创建数据库记录
- 即使用户不保存，记录也存在
- 未来可以添加"草稿"状态来优化

### 错误处理
- AI 生成失败时，返回预览页面
- 保存失败时，保留用户输入的内容
- 网络错误时，显示友好提示
