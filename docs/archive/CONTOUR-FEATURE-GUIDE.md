# 粒子轮廓功能使用指南

## 功能说明

粒子轮廓功能会将生物的图片转换为动态的粒子动画效果，让生物详情页面更加生动有趣。

## 工作原理

### 智能加载机制

系统采用了**智能按需加载**的设计：

1. **新创建的生物**：在创建时自动提取轮廓数据并保存到数据库
2. **现有的生物**：第一次查看详情时，自动从图片提取轮廓数据
3. **已有轮廓的生物**：直接使用缓存的轮廓数据，无需重新计算

### 优势

✅ **兼容现有数据**：不需要重新创建生物，现有生物也能看到粒子效果  
✅ **性能优化**：轮廓数据会被缓存，第二次查看时立即显示  
✅ **用户体验好**：首次加载时显示加载动画，用户知道系统在工作  
✅ **容错性强**：如果提取失败，会显示友好的错误提示

## 如何查看效果

### 方法 1：查看现有生物（推荐）

1. 启动后端服务器：
   ```bash
   cd backend
   npm run dev
   ```

2. 启动前端服务器：
   ```bash
   cd frontend
   npm run dev
   ```

3. 在浏览器中打开应用：`http://localhost:5173`

4. 点击任意生物卡片查看详情

5. 右侧会显示：
   - 如果是第一次查看：显示"加载轮廓数据中..."，然后显示粒子动画
   - 如果已经加载过：直接显示粒子动画

### 方法 2：性能测试页面

查看纯粹的粒子动画效果和性能指标：

1. 启动前端服务器
2. 访问：`http://localhost:5173/test-particle-performance.html`
3. 可以调整粒子数量和情绪值，实时查看效果

## 技术细节

### 后端 API

**新增端点**：`GET /api/creatures/:id/contour`

- 如果生物已有轮廓数据，直接返回
- 如果没有，从图片提取轮廓并保存到数据库
- 返回格式：
  ```json
  {
    "points": [
      { "x": 0.5, "y": 0.3 },
      { "x": 0.6, "y": 0.4 },
      ...
    ],
    "boundingBox": {
      "minX": -1.0,
      "maxX": 1.0,
      "minY": -1.0,
      "maxY": 1.0
    }
  }
  ```

### 前端实现

**CreatureInfoPanel 组件**：
- 使用 `useEffect` 监听生物变化
- 如果没有轮廓数据，自动调用 API 获取
- 显示加载状态和错误处理
- 获取成功后渲染 ParticleOutlineViewer 组件

### 性能优化

1. **轮廓点数优化**：目标 200 个点（范围 100-300）
2. **前端粒子限制**：最多 300 个粒子
3. **动画参数调优**：减少振幅，提高流畅度
4. **数据库缓存**：轮廓数据保存后不再重复计算

## 常见问题

### Q: 为什么第一次查看生物时需要等待？
A: 系统正在从图片提取轮廓数据。这个过程通常在 1-3 秒内完成，之后会被缓存。

### Q: 如果轮廓提取失败会怎样？
A: 会显示友好的错误提示，不会影响其他功能的使用。

### Q: 现有的生物需要重新创建吗？
A: 不需要！现有生物第一次查看时会自动生成轮廓数据。

### Q: 粒子动画会影响性能吗？
A: 不会。我们优化了粒子数量和动画参数，确保在大多数设备上保持 60 FPS。
