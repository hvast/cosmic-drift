# 生物创建模块实现文档

## 概述

本模块实现了星际漂流计划（Cosmic Drift）的核心功能之一：生物创建系统。用户可以通过绘制或上传图片来创建数字生命，AI会自动生成完整的生物档案。

## 已实现的功能

### 前端组件

#### 1. CreatureCanvas (画布绘图组件)
- **位置**: `frontend/src/components/CreatureCanvas.tsx`
- **功能**:
  - 使用 Fabric.js 实现的交互式画布
  - 支持自由绘图、颜色选择、笔刷大小调整
  - 橡皮擦工具
  - 撤销功能
  - 清空画布
  - 导出为 PNG 图片

#### 2. ImageUploader (图片上传组件)
- **位置**: `frontend/src/components/ImageUploader.tsx`
- **功能**:
  - 支持拖拽和点击上传
  - 图片预览
  - 格式验证（PNG, JPG, WebP）
  - 尺寸验证（100x100 到 4096x4096 像素）
  - 文件大小限制（最大 5MB）
  - 友好的错误提示

#### 3. ProfileEditor (档案编辑器组件)
- **位置**: `frontend/src/components/ProfileEditor.tsx`
- **功能**:
  - 展示 AI 生成的完整生物档案
  - 支持编辑名称和背景故事
  - 显示物种、性格特征、栖息地
  - 情绪值可视化
  - 发布到宇宙功能
  - 优美的动画效果（使用 Framer Motion）

#### 4. CreatePage (创建页面)
- **位置**: `frontend/src/pages/CreatePage.tsx`
- **功能**:
  - 创建方式选择（绘制 vs 上传）
  - 集成画布和上传组件
  - AI 生成进度提示
  - 档案编辑和发布流程
  - 完整的用户体验流程

### 后端服务

#### 1. ImageAnalysisService (图像分析服务)
- **位置**: `backend/src/services/ImageAnalysisService.ts`
- **功能**:
  - 使用 OpenAI Vision API 分析图像
  - 提取视觉特征（主色调、风格、复杂度）
  - 重试机制（最多 3 次）
  - 降级方案（当 AI 服务不可用时）
  - 基于视觉特征生成初步档案建议

#### 2. ProfileGeneratorService (档案生成服务)
- **位置**: `backend/src/services/ProfileGeneratorService.ts`
- **功能**:
  - 使用 GPT-4 生成完整生物档案
  - 生成物种名称、性格特征、栖息地描述
  - 生成富有诗意的背景故事
  - 支持用户自定义（名称、故事片段）
  - 档案验证和格式化
  - 重试机制和降级方案

#### 3. StorageService (存储服务)
- **位置**: `backend/src/services/StorageService.ts`
- **功能**:
  - 保存 base64 图片到本地文件系统
  - 图片格式验证
  - 文件大小验证
  - 生成唯一文件名
  - 返回公开访问 URL

#### 4. CreatureService (生物服务)
- **位置**: `backend/src/services/CreatureService.ts`
- **功能**:
  - 协调整个生物创建流程
  - 并行处理图像分析和存储
  - 性能监控（确保 10 秒内完成）
  - CRUD 操作（创建、读取、更新、删除）
  - 分页查询
  - 随机遇见功能

### 数据层

#### 1. Creature Model
- **位置**: `backend/src/models/Creature.ts`
- **字段**:
  - id, name, species, personality
  - habitat, backstory, imageUrl
  - creatorId, adopterId, status
  - emotionValue, createdAt, adoptedAt

#### 2. CreatureRepository
- **位置**: `backend/src/repositories/CreatureRepository.ts`
- **功能**:
  - 数据库 CRUD 操作
  - 按创建者查询
  - 查询漂流中的生物
  - 分页支持
  - 统计功能

### API 端点

#### 1. CreatureController
- **位置**: `backend/src/controllers/CreatureController.ts`
- **端点**:
  - `POST /api/creatures` - 创建生物
  - `GET /api/creatures` - 获取所有生物（分页）
  - `GET /api/creatures/:id` - 获取单个生物
  - `GET /api/creatures/my/list` - 获取用户的生物
  - `GET /api/creatures/random` - 随机遇见
  - `PATCH /api/creatures/:id` - 更新生物
  - `DELETE /api/creatures/:id` - 删除生物

#### 2. 路由配置
- **位置**: `backend/src/routes/creatures.ts`
- **特性**:
  - 公开路由（查看生物）
  - 受保护路由（创建、编辑、删除）
  - 输入验证中间件

### 类型定义

#### 前端类型
- **位置**: `frontend/src/types/creature.ts`
- 包含所有生物相关的 TypeScript 接口

#### 后端类型
- **位置**: `backend/src/types/creature.ts`
- 与前端保持一致的类型定义

## 技术栈

### 前端
- React 18 + TypeScript
- Fabric.js (画布绘图)
- Framer Motion (动画)
- TailwindCSS (样式)
- Axios (HTTP 请求)

### 后端
- Node.js + Express + TypeScript
- PostgreSQL (数据存储)
- OpenAI API (GPT-4 + Vision)
- Axios (HTTP 客户端)
- Joi (输入验证)

## 性能优化

1. **并行处理**: 图像分析和存储同时进行
2. **重试机制**: AI 服务失败时自动重试（最多 3 次）
3. **降级方案**: AI 不可用时使用预设模板
4. **性能监控**: 记录处理时间，警告超过 10 秒的请求

## 安全特性

1. **输入验证**: 
   - 图片格式和大小验证
   - 请求体验证（使用 Joi）
2. **认证授权**:
   - JWT token 认证
   - 用户只能编辑/删除自己的生物
3. **文件上传安全**:
   - 文件类型白名单
   - 大小限制（5MB）
   - 唯一文件名生成

## 使用流程

1. 用户访问创建页面
2. 选择创建方式（绘制或上传）
3. 完成图片创作/上传
4. 点击"生成档案"
5. AI 分析图像并生成档案（约 5-10 秒）
6. 用户查看并可编辑档案
7. 点击"发布到宇宙"完成创建
