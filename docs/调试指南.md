# 🐛 调试指南

## 常见错误及解决方案

### 1. 端口被占用 (EADDRINUSE)

**错误信息**:
```
Error: listen EADDRINUSE: address already in use :::3001
```

**原因**: 端口 3001 已被占用

**解决方案**:

#### 方法 1: 找到并关闭占用端口的进程
```powershell
# 查找占用 3001 端口的进程
netstat -ano | findstr :3001

# 会显示类似：
# TCP    0.0.0.0:3001    0.0.0.0:0    LISTENING    12345
# 最后的数字(12345)是进程ID(PID)

# 关闭该进程
taskkill /PID 12345 /F
```

#### 方法 2: 修改端口
编辑 `backend/.env`:
```env
PORT=3002  # 改成其他端口
```

然后前端也要改 `frontend/.env`:
```env
REACT_APP_API_URL=http://localhost:3002
```

### 2. 数据库连接失败

**错误信息**:
```
❌ MySQL connection error
```

**解决方案**:

#### 检查 MySQL 是否运行
```powershell
# 检查 MySQL 服务状态
Get-Service -Name MySQL*

# 启动 MySQL 服务
Start-Service MySQL80  # 或你的 MySQL 服务名
```

#### 检查数据库配置
编辑 `backend/.env`:
```env
DB_HOST=localhost
DB_PORT=3306
DB_NAME=cosmic_drift
DB_USER=root
DB_PASSWORD=你的密码
```

#### 测试数据库连接
```bash
mysql -u root -p
USE cosmic_drift;
SHOW TABLES;
```

### 3. 前端无法连接后端

**错误信息**:
```
Network Error
ECONNREFUSED
```

**解决方案**:

#### 检查后端是否运行
```powershell
# 测试后端
curl http://localhost:3001/api/creatures
# 或
Invoke-WebRequest http://localhost:3001/api/creatures
```

#### 检查 CORS 配置
确保 `backend/src/index.ts` 中有：
```typescript
app.use(cors({
  origin: process.env.CORS_ORIGIN || 'http://localhost:3000'
}));
```

### 4. 401 Unauthorized 错误

**错误信息**:
```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Not authenticated"
  }
}
```

**原因**: Token 无效或过期

**解决方案**:
1. 重新登录获取新 token
2. 检查 Authorization header 格式：`Bearer <token>`
3. 确保 token 没有多余的空格或换行

### 5. 404 Not Found 错误

**错误信息**:
```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Creature not found"
  }
}
```

**原因**: 资源不存在

**解决方案**:
1. 检查 ID 是否正确
2. 确认资源已创建
3. 查看数据库中是否有该记录

## 🔍 调试技巧

### 1. 查看后端日志

后端终端会显示所有请求和错误：
```
Server running on port 3001
✅ MySQL database connected successfully
POST /api/auth/register 201 - 234ms
GET /api/creatures 200 - 45ms
```

**关键信息**:
- HTTP 方法和路径
- 状态码 (200=成功, 400=客户端错误, 500=服务器错误)
- 响应时间

### 2. 使用浏览器开发者工具

1. 打开前端 (http://localhost:3000)
2. 按 F12 打开开发者工具
3. 切换到 **Network** 标签
4. 执行操作（如登录、创建生物）
5. 查看请求详情：
   - Request Headers
   - Request Payload
   - Response
   - Status Code

### 3. 查看数据库数据

```bash
mysql -u root -p
USE cosmic_drift;

# 查看所有表
SHOW TABLES;

# 查看用户
SELECT * FROM users;

# 查看生物
SELECT * FROM creatures;

# 查看最近的记录
SELECT * FROM creatures ORDER BY created_at DESC LIMIT 5;

# 查看特定用户的生物
SELECT * FROM creatures WHERE creator_id = 'user-id-here';
```

### 4. 使用 console.log 调试

在代码中添加日志：

**后端**:
```typescript
console.log('收到请求:', req.body);
console.log('查询结果:', result);
console.log('错误信息:', error);
```

**前端**:
```typescript
console.log('发送数据:', data);
console.log('响应结果:', response);
console.error('错误:', error);
```

### 5. 使用 VS Code 调试器

#### 后端调试配置 (`.vscode/launch.json`):
```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug Backend",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "dev"],
      "cwd": "${workspaceFolder}/backend",
      "console": "integratedTerminal"
    }
  ]
}
```

然后：
1. 在代码中设置断点（点击行号左侧）
2. 按 F5 启动调试
3. 执行操作触发断点
4. 查看变量值、调用栈等

## 📊 API 测试工具

### 1. VS Code REST Client

安装插件后，打开 `api-tests.http`，点击 "Send Request"

### 2. Postman

1. 导入 `api-tests.http`
2. 设置环境变量
3. 运行测试集合

### 3. curl 命令

```bash
# GET 请求
curl http://localhost:3001/api/creatures

# POST 请求
curl -X POST http://localhost:3001/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@example.com","password":"Test123456"}'

# 带 Token 的请求
curl http://localhost:3001/api/creatures/my/list \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

### 4. PowerShell

```powershell
# GET 请求
Invoke-RestMethod -Uri "http://localhost:3001/api/creatures"

# POST 请求
$body = @{
    username = "test"
    email = "test@example.com"
    password = "Test123456"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:3001/api/auth/register" `
  -Method POST `
  -Body $body `
  -ContentType "application/json"
```

## 🚨 紧急问题排查清单

遇到问题时，按顺序检查：

- [ ] 1. 后端是否在运行？
  ```powershell
  # 检查进程
  Get-Process node
  ```

- [ ] 2. 端口是否被占用？
  ```powershell
  netstat -ano | findstr :3001
  ```

- [ ] 3. MySQL 是否运行？
  ```powershell
  Get-Service MySQL*
  ```

- [ ] 4. 数据库是否存在？
  ```bash
  mysql -u root -p
  SHOW DATABASES;
  ```

- [ ] 5. 环境变量是否配置？
  - 检查 `backend/.env`
  - 检查 `frontend/.env`

- [ ] 6. 依赖是否安装？
  ```bash
  cd backend && npm install
  cd frontend && npm install
  ```

- [ ] 7. 数据库迁移是否运行？
  ```bash
  npm run migrate
  ```

## 📝 错误日志收集

遇到无法解决的问题时，收集以下信息：

1. **错误信息**
   - 完整的错误堆栈
   - 错误发生的时间

2. **环境信息**
   - Node.js 版本: `node --version`
   - npm 版本: `npm --version`
   - MySQL 版本: `mysql --version`

3. **后端日志**
   - 复制后端终端的输出

4. **前端日志**
   - 浏览器 Console 的错误信息
   - Network 标签的请求详情

5. **配置文件**
   - `backend/.env` (隐藏敏感信息)
   - `frontend/.env`

## 💡 最佳实践

1. **开发时保持终端可见**
   - 随时查看日志输出
   - 及时发现错误

2. **使用有意义的日志**
   ```typescript
   console.log('[Auth] 用户登录:', email);
   console.log('[Creature] 创建生物:', creatureData);
   ```

3. **错误处理要完整**
   ```typescript
   try {
     // 操作
   } catch (error) {
     console.error('详细错误:', error);
     // 返回友好的错误信息
   }
   ```

4. **定期清理测试数据**
   ```sql
   DELETE FROM creatures WHERE name LIKE 'test%';
   DELETE FROM users WHERE email LIKE 'test%';
   ```

5. **使用版本控制**
   ```bash
   git status
   git diff
   git checkout -- file  # 恢复文件
   ```
