# 🪐 星际漂流计划 - 快速启动指南

## 🚀 一键启动

### Windows 用户

#### 方式 1: 使用批处理脚本（推荐）
双击运行 `scripts/快速启动.bat`，然后选择启动方式：
- 选项 1: 启动完整开发环境（前端 + 后端）
- 选项 2: 仅启动前端
- 选项 3: 仅启动后端
- 选项 4: 安装所有依赖
- 选项 5: 运行数据库迁移

#### 方式 2: 使用命令行
```bash
# 启动完整开发环境
scripts\start-dev.bat

# 或使用 PowerShell
.\scripts\start-dev.ps1
```

#### 方式 3: 使用 npm 命令
```bash
# 首次运行：安装所有依赖
npm run install:all

# 启动完整开发环境
npm run dev

# 仅启动前端
npm run dev:frontend

# 仅启动后端
npm run dev:backend
```

## 📦 首次运行步骤

### 1. 安装依赖
```bash
# 方式 1: 使用脚本
scripts\install-all.bat

# 方式 2: 使用 npm
npm run install:all
```

### 2. 配置环境变量

**后端配置** (`backend/.env`):
```env
# 复制 backend/.env.example 并修改以下配置
PORT=5000
NODE_ENV=development

# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_NAME=cosmic_drift
DB_USER=your_username
DB_PASSWORD=your_password

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379

# JWT 密钥
JWT_SECRET=your-secret-key-here
JWT_REFRESH_SECRET=your-refresh-secret-here

# AI 服务配置
QWEN_API_KEY=your-qwen-api-key
```

**前端配置** (`frontend/.env`):
```env
# 复制 frontend/.env.example 并修改
REACT_APP_API_URL=http://localhost:5000
REACT_APP_WS_URL=http://localhost:5000
```

### 3. 启动数据库服务

确保 PostgreSQL 和 Redis 已安装并运行：

**PostgreSQL:**
```bash
# 创建数据库
createdb cosmic_drift
```

**Redis:**
```bash
# 启动 Redis 服务
redis-server
```

### 4. 运行数据库迁移
```bash
npm run migrate
```

### 5. 启动开发服务器
```bash
# 方式 1: 使用脚本
scripts\start-dev.bat

# 方式 2: 使用 npm
npm run dev
```

## 🌐 访问地址

启动成功后，访问以下地址：

- **前端**: http://localhost:3000
- **后端 API**: http://localhost:5000

## 🐛 常见问题

### 1. 端口被占用
如果 3000 或 5000 端口被占用，可以修改：
- 前端: 在 `frontend/.env` 中设置 `PORT=3001`
- 后端: 在 `backend/.env` 中设置 `PORT=5001`

### 2. 依赖安装失败
```bash
# 清理缓存后重新安装
npm cache clean --force
npm run install:all
```

### 3. 数据库连接失败
- 检查 PostgreSQL 是否运行
- 验证 `backend/.env` 中的数据库配置
- 确保数据库已创建

### 4. Redis 连接失败
- 检查 Redis 是否运行
- 验证 `backend/.env` 中的 Redis 配置
