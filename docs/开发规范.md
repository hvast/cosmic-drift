# 开发规范

## 代码风格

### TypeScript/JavaScript

- 使用 TypeScript 进行类型安全开发
- 使用 ESLint 进行代码检查
- 使用 Prettier 进行代码格式化
- 遵循 Airbnb JavaScript Style Guide

### 命名规范

- **文件名**: 使用 PascalCase（组件）或 camelCase（工具函数）
  - 组件: `CreatureCanvas.tsx`
  - 服务: `imageAnalysisService.ts`
  - 工具: `formatDate.ts`

- **变量名**: 使用 camelCase
  ```typescript
  const userName = 'Alice';
  const creatureList = [];
  ```

- **常量**: 使用 UPPER_SNAKE_CASE
  ```typescript
  const MAX_FILE_SIZE = 5 * 1024 * 1024;
  const API_BASE_URL = 'http://localhost:5000';
  ```

- **类名/接口**: 使用 PascalCase
  ```typescript
  class CreatureService {}
  interface CreatureProfile {}
  ```

- **函数名**: 使用 camelCase，动词开头
  ```typescript
  function createCreature() {}
  function getUserById() {}
  ```

### 注释规范

- 使用 JSDoc 格式注释函数和类
- 复杂逻辑添加行内注释
- 避免无意义的注释

```typescript
/**
 * 创建新的生物
 * @param imageData - Base64 编码的图片数据
 * @param customization - 用户自定义信息
 * @returns 创建的生物档案
 */
async function createCreature(
  imageData: string,
  customization?: CreatureCustomization
): Promise<CreatureProfile> {
  // 实现逻辑
}
```

## Git 提交规范

### Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整（不影响功能）
- `refactor`: 重构代码
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具相关

### 示例

```
feat(creature): 添加画布绘图功能

- 集成 Fabric.js
- 实现画笔、橡皮擦工具
- 添加颜色选择器

Closes #123
```

## 分支管理

### 分支命名

- `main`: 主分支，稳定版本
- `develop`: 开发分支
- `feature/xxx`: 功能分支
- `fix/xxx`: 修复分支
- `hotfix/xxx`: 紧急修复分支

### 工作流程

1. 从 `develop` 创建功能分支
2. 开发完成后提交 PR
3. Code Review 通过后合并到 `develop`
4. 定期从 `develop` 合并到 `main`

## API 设计规范

### RESTful API

- 使用名词复数形式
- 使用 HTTP 动词表示操作
- 使用合适的状态码

```
GET    /api/creatures       # 获取列表
GET    /api/creatures/:id   # 获取单个
POST   /api/creatures       # 创建
PATCH  /api/creatures/:id   # 更新
DELETE /api/creatures/:id   # 删除
```

### 响应格式

成功响应：
```json
{
  "success": true,
  "data": { ... }
}
```

错误响应：
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input",
    "details": { ... }
  }
}
```

## 错误处理

### 前端

```typescript
try {
  const result = await api.createCreature(data);
  // 处理成功
} catch (error) {
  if (error.response) {
    // API 错误
    showError(error.response.data.error.message);
  } else {
    // 网络错误
    showError('网络连接失败，请重试');
  }
}
```

### 后端

```typescript
try {
  // 业务逻辑
} catch (error) {
  logger.error('Error creating creature:', error);
  res.status(500).json({
    success: false,
    error: {
      code: 'INTERNAL_ERROR',
      message: 'Failed to create creature'
    }
  });
}
```

## 测试规范

### 单元测试

- 测试文件命名: `*.test.ts` 或 `*.spec.ts`
- 测试覆盖率目标: 80%
- 使用 Jest 作为测试框架

```typescript
describe('CreatureService', () => {
  it('should create creature successfully', async () => {
    const result = await creatureService.create(mockData);
    expect(result).toBeDefined();
    expect(result.name).toBe('Test Creature');
  });
});
```

## 性能优化

### 前端

- 使用 React.memo 避免不必要的重渲染
- 使用 useMemo 和 useCallback 优化计算
- 图片懒加载
- 代码分割（React.lazy）

### 后端

- 使用数据库索引
- 实现缓存策略（Redis）
- 避免 N+1 查询
- 使用连接池

## 安全规范

### 前端

- 避免 XSS 攻击（使用 React 自动转义）
- 敏感信息不存储在 localStorage
- HTTPS 传输

### 后端

- 输入验证（使用 Joi）
- SQL 注入防护（使用参数化查询）
- 密码加密（bcrypt）
- JWT token 认证
- Rate limiting

## 文档规范

### README

每个模块应包含：
- 功能说明
- 使用方法
- API 文档
- 示例代码

### 代码注释

- 复杂逻辑必须注释
- 公共 API 必须有 JSDoc
- TODO 注释格式: `// TODO: 描述`

### 命名规范

- 敏感信息不提交到 Git
